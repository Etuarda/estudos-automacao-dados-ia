# Miniguia de Estudo: Engenharia de Prompts para Backend

## 1. Conceitos Principais

- **IA (Inteligência Artificial):** Área da computação dedicada a criar sistemas capazes de executar tarefas que normalmente exigiriam inteligência humana, como reconhecimento de padrões, tomada de decisão e processamento de linguagem natural.

- **LLM (Large Language Model):** Modelo de linguagem treinado em grandes volumes de texto, capaz de gerar, resumir e interpretar linguagem natural. Exemplos: GPT-4, PaLM, Llama.

- **Engenharia de Prompts:** Técnica de criar instruções claras e contextuais para obter melhores respostas de IAs generativas. Envolve refinamento iterativo e análise crítica dos resultados.

## 2. Elementos de um Bom Prompt

- Clareza no objetivo
- Contexto técnico (ex: linguagem, framework)
- Exemplos práticos
- Solicitação de explicações passo a passo

## 3. Exemplos de Prompts Reutilizáveis

- Explique [conceito] com foco em backend, incluindo definição, exemplo prático e erro comum.
- Compare [tecnologia A] e [tecnologia B] considerando contexto de projeto júnior.
- Revise este resumo técnico e aponte inconsistências, lacunas e sugestões de melhoria.
- Transforme este conteúdo em um plano de estudo de 5 dias com ordem progressiva de aprendizado.
- Gere exemplos de código para [tecnologia] aplicados a APIs REST.
- Liste boas práticas para [tema] no contexto de backend.

## 4. Glossário Rápido

- **Prompt:** Instrução fornecida ao modelo de IA.
- **Contexto:** Informação adicional que orienta a resposta.
- **Alucinação:** Resposta incorreta apresentada com aparência de confiança.
- **Refinamento iterativo:** Melhoria progressiva do prompt com base no resultado anterior.

## 5. Dicas de Uso

- Sempre revise e valide as respostas da IA.
- Refine o prompt se a resposta não for satisfatória.
- Use exemplos e contexto para obter respostas mais úteis.
